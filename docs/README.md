# Practice notes

This repository was prepared as part of production practice work and developed toward MVP state.

## Practice tasks addressed

1. Open at least 3 issues in the project
2. Close issues through pull requests
3. Bring the project to the level of a minimally working prototype

## MVP interpretation for this project

For this repository, MVP means:
- a reproducible repository structure
- working evaluation scripts
- saved outputs in JSON / CSV form
- documented setup steps
- at least one functioning simulation stage

## Main improvement directions

- normalize configuration and model setup
- make statistical helpers robust
- stabilize and extend the ABM simulation
